¿Qué es OTS y cómo verificar?

Cada checkpoint tiene un Merkle root (huella digital). OTS lo envía a 4 calendarios públicos que lo incluyen en una transacción Bitcoin. Cualquier auditor puede verificar que el dato existía antes de cierto bloque sin depender de Centinel.

Calendarios usados (gratuitos, sin API key):

  • alice.btc.calendar.opentimestamps.org
  • bob.btc.calendar.opentimestamps.org
  • finney.calendar.eternitywall.com
  • ots.btc.catallaxy.com

Para verificar una prueba OTS:

  1. Descarga el archivo .ots del checkpoint
  2. pip install opentimestamps-client && ots verify archivo.ots
  3. O usa opentimestamps.org online

Si OTS no está disponible, los snapshots continúan normalmente y las raíces pendientes se reintentan en el próximo ciclo.

Manual ⓘ
Guía rápida del panel de administración

Configuración avanzada

Los parámetros de operación se configuran desde la sección Presets. Carga un preset para ajustar el intervalo de captura, requests/hora, modos electoral y seguro.

Para configuración detallada, expande Configuración avanzada en la sección Presets: 5 tabs con acceso a endpoints individuales, watchdog, rate limiter, red y reglas de detección.

El mapa de endpoints

Muestra los 18 departamentos + NACIONAL de Honduras como puntos geoposicionados. El color indica el estado del endpoint configurado. Click en cualquier punto para editar la URL o probar el endpoint.

Cómo se guardan los cambios

Cada cambio se guarda automáticamente 1.5 segundos después de modificarlo.
1. Modifica cualquier control — el cambio se envía a GitHub automáticamente.
2. La primera vez se te pide el GitHub PAT (solo en sessionStorage, desaparece al cerrar el browser).
3. El sistema hace commit a config/prod/ vía GitHub API.
4. Los cambios toman efecto en el próximo ciclo (~15 min).

Animal mode

NORMAL: operación estándar.
CAUTION: el sistema detectó condiciones adversas, reduce agresividad.
SURVIVAL: modo mínimo, solo captura esencial, máxima resiliencia.

Qué hacer si un endpoint queda rojo

1. Click en el punto en el mapa → "Probar ahora" para verificar si responde.
2. Si no responde: verifica la URL con el proveedor CNE.
3. Actualiza la URL en el popup → "Guardar cambio" → se guarda automáticamente.
4. Si es un fallo temporal, el circuit breaker se recupera automáticamente.

Recuperar configuración anterior

Todo cambio es un commit en config/prod/. Para revertir: ve a GitHub → config/prod/endpoints.yaml → History → elige el commit anterior → "Revert" o copia el contenido manualmente.

⚠ ADVERTENCIA DE SEGURIDAD OPERATIVA

Estás por desactivar los límites de cortesía digital de fábrica de CENTINEL.
Por diseño original, este software opera bajo un Modo Electoral Seguro que respeta las directivas del servidor (robots.txt), los frenos de saturación (Retry-After) y se identifica honestamente ante la infraestructura auditada.

Al proceder con esta acción, declaras conscientemente lo siguiente:

Autonomía: Entiendes que estás forzando la herramienta más allá de los límites éticos y técnicos establecidos por su creador (máx 480 req/h, mín 2s intervalo, burst 8).
Responsabilidad: Asumes de forma exclusiva y total cualquier consecuencia técnica, legal o administrativa derivada del aumento en la intensidad de las peticiones.
Desvinculación: Eximes a la suite CENTINEL y a sus desarrolladores de cualquier implicación por el uso derivado de esta sesión.

Para desbloquear los límites, confirma la acción. Tu aceptación se grabará de forma permanente e inmutable mediante un commit firmado en el historial del repositorio.
